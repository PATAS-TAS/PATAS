# Outcome

PATAS now has a primary-source-checked GitHub dependency landscape. The first
post-buyer benchmark should use scikit-learn as its common spine and test only
bounded challengers for MinHash blocking, semantic retrieval, interpretable
rule hypotheses, and local replay. Large ML/data platforms and drop-in spam
systems are explicitly excluded from the first pilot.

# Evidence

- More than thirty active, archived, overlapping, and adjacent repositories
  were checked through current repository metadata, READMEs, implementation
  files, and recent commit history.
- The audit found a Python compatibility decision: current scikit-learn and
  RapidFuzz main branches require Python 3.11 while PATAS declares Python 3.10.
- The audit found reproducibility risks in persisted MinHash scheme versions
  and HDBSCAN parameter semantics.
- The product-specific rights manifest, evidence card, typed rule compiler,
  replay contract, and artifact hashes remain PATAS-owned work.

# Remaining

No dependency is adopted and no v3 runtime is started until a qualified buyer
makes a paid commitment and the metadata-only data preflight passes.

# Next trigger

A paid qualified pilot supplies an approved data shape, triggering a disposable
baseline-versus-challenger benchmark and an explicit Python runtime decision.
